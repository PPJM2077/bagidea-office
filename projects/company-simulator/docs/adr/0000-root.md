# ADR-0000: root module (app layer)

> **Status:** Accepted
> **Date:** 2026-07-07
> **Owner:** architect

---

## Context

This ADR records the existence and position of the **`root`** module so the
`check-adr` gate has a documented decision for it. `root` is the project-root
manifest (`module.json` at the repository root) -- it declares the whole
`company-simulator` project as a single deliverable app-layer unit, owned by
the architect with the CTO as reviewer.

- Package: `@company-simulator/root`
- Declared upstream dependencies:
- (none -- the root manifest aggregates the app; per-module deps live in each `<layer>/<name>/module.json`)
- Health targets:
- maxPublicExports: 0
- maxCyclomaticComplexity: 12
- testCoverageRequired: 80

---

## Decision

Establish the project-root `module.json` as a module in the **app** layer with
the health targets recorded above. The root manifest is the source of truth for
the project's identity (`@company-simulator/root`), version, and ownership;
per-module public surfaces and dependency graphs live in each
`<layer>/<name>/module.json`.

Any change to the root manifest's layer, version, or ownership requires
updating this ADR (or a superseding ADR) so the decision record stays in sync.

---

## Consequences

### Positive
- The root project manifest is a first-class module: it is counted by the
  validation gate (15 modules, not 14) and carries the same ADR obligation as
  any layer module.
- `check-adr` can enforce that the root manifest is not changed silently.

### Negative
- One more ADR file to keep in sync with the root `module.json`.

### Trade-offs
- Treating the root manifest as a module makes the module count honest (15)
  rather than silently excluding it; the cost is one extra ADR.

## Options Considered

| Option | Pros | Cons | Verdict |
|--------|------|------|---------|
| **Root is a module with its own ADR** | honest module count; gate not vacuous | one extra ADR | ✅ |
| Exclude root from the module count | fewer files | gate under-counts (14 not 15); silent | ❌ |
| Blanket exemption for root | no files needed | gate becomes vacuous | ❌ |
